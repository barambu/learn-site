### 9
Отступы и рамки
- padding - внутренний отступ от контейнера
- border - рамка контейнера
- margin - внешний отступ за контейнером
- top - верх, bottom - низ
Два варианта указания этих характеристик:
```
div {
    margin-top: 70px;
    padding-top: 30px;
    border-width: 3px;
    border-color: black;
    border-style: ridge;
    background-color: pink;
}
```
В данном варианте характеристики margin прописываются по часовой стрелке (верх-право-низ-лево), в border пишетс сначала толщина рамки, затем её вид, потом цвет:
```
div {
    margin: 70px 50px 30px 10px;
    padding: 30px 30px 30px 0px;
    border: 3px ridge black ;
    background-color: pink;
}
```
### 10
Есть 2 вида эл-тов: строчные и блочные.
- Строчные занимают то пространство, которое занимет их контент (например, их текст)
- В строчные эл-ты нельзя вкладыват блочные
- У строчных эл-тов нельзя сделать вертикальные отступы с помощью margin и заавать ширину с пом. width
- Блочные эл-ты занимат полностью всю доступную ширину, всегда начинаются с новой строки
- В блочные эл-ты можно вставлять и блочные, и строчные эл-ты (в строчный нельзя вставить блочный)
- Любой строчный и блочный эл-т можно превратить в строчный или блочный эл-т, просто прописав ему `display: inline` или `display: block`(удобно для строчки, чтобы менять её расп-е)
- Можно сделать нечто среднее, сохранив всё лучшее от строки и блока - прописывает `display: inline block`.

`* {box-sizing: border-box}` - любой элемент состоит из заданных нами размеров + все паддинги, марджины. Чтобы задать точный и правильный размер эл-та, используем данную комаду в самом начале документа style.css - она выыравнивает наш элемент по центру (например, заданная рамка (бордер) встанет ровно по центру нашего родительского э-та). Селектор * означает, что данная команда применяется ко всем эл-там. Это стоит прописать в начале документа сразу, в начале создания сайта.

### 21
- learn/javascript.ru/css-units - о том, какие единицы измерения есть в CSS.
% - относительная единица измерения. Нужно зафиксировать эл-т в одном положении при разных масштабах экрана (настроить эл-т таким образом, чтобы его размеры слева или сверху менялись относительно размеров в ширину нашего экрана). При расположении эл-та мы задаём ему нужную ширину и высоту от краёв в пикселях. Для фиксации нужно высчитать эти характеристики в % - `данную ширину/высоту элемента РАЗДЕЛИТЬ на всю ширину/высоту места, где размещён элемент И УМНОЖИТЬ на 100 `. Далее - вписать эти характеристики в код (на места отступов - top bottom left right).
Пример:
```
.product-img__block {
    width: 420px;
    height: 190px;
    background: linear-gradient(180deg, rgba(244, 213, 178, 0.8)0%, rgba(234, 156, 65, 0.8)100%);
    position: absolute;
    top: 54%;
    left: 50%;
    padding-top: 15px;
}
```

### 25
У каждого эл-та есть свои стили по умолчанию (в частности, марджины). Есть 2 проблемы:
- разное отображение эл-тов в разных браузерах
- бесконечные стандартные стили в эл-тах

 У всего подобного, есть 2 варианта решения - нормализаторы стилей css:
 - `normalize.css` - перейдя по ссылке, попадём на текстовую страницу, где вылезет код с нужными параметрами. Чтобы подключить этот файл к вёрстке, можно создать отдельный лист со стилями и прописываем сслыку в наш html файл (путь `link`). А можно иначе - скопировать ссылку на эту страницу с текстом кода, создать `link` над(!!!) нашим файло css и вставить в`href` скопированную ссылку. Файл отвечает за то, чтбы на разных браузерах всё выглядело одинаково.
 - `reset.css` - если создать новый эл-т на сайте, когда подключен только нормалайз.цсс, то стандартные отступы не обнулятся. Нужно добавить в код ещё и reset.css. Тут прописаны все сбросы стандартных стилей, которые мешают нам в работе. Устанавливается НАД файлом normalize.css.

- задавать каждому элементу шрифты вручную (из уже имеющихся на нащем компьютере), используя команды `font-family`(названия шрифтов),`font-weight` (толщина шрифта - начертание) 

![wmplayer_5yQoSSrpH6](https://user-images.githubusercontent.com/109949290/182657942-d33ba14e-f13c-4732-a945-e38c39ef1d7a.png)

- через `fonts.google.com`: зайти на сайт-выбрать нужные стили (нажать на плюик слева)-в правом верхнем углу открыть последний значок-скопировать код для link - вставить в наш файл html. Активировав их, прописывать также в каждом нужном элементе свой шрифт (код для css можно скопировать там же + добавить хар-ки начертания).

![wmplayer_RvqoZtltHA](https://user-images.githubusercontent.com/109949290/182659751-7b2f4a23-e95d-49ba-aac2-cedec8763ee7.png)

- скачать шрифты с сайта в виде документа (download all). Затем с помощью команды `@font-face` подключить эти шрифты в файле css. Через `font-family: 'название'` указываем название сайта (своё) - `scr:local(настоящее название)` - `src: url(../fonts/shrift.ttf)` прокладываем путь - указываем свойства для нужного эл-та чере название шрифта. Пример:
```
@font-face {
    font-family: 'ROBOTO';
    src:local('Roboto');
    src: url(../fonts/Roboto-Regular.ttf);
    
}

h1 {
    font-family: 'ROBOTO';
}
```
- скачать нескольrо форматов наших шрифтов с сайта `webfonts.pro` (eot, woff, ttf) -> скопировать код для css с сайта и вставить его ->подправить пути, указав папку (лучше писать вручную!!!, чтобы не стёрся текст) - в команду `font-family: 'название своё'` вписать своё название -> добавить команду `src:local('Roboto')`. Затем мы обращаемся к тегу html и прописываем ему команду со своим наз-ем шрифта -`font-family: 'название своё'`. Добавляя неск. шрифтов, можно указать разную высоту, и обращаясь к элементам указывать высоту шрифта, чтобы было понятно, какой именно нужен. Либо можно задать разные названия (свои) в фонт-фэмили и добавлять это свойство. Пример:
```
html {
    font-family: 'Roboto!';   
}

@font-face {
    font-family: 'Roboto!';
    src: url('../fonts/RobotoCondensed-Regular.eot');
    src: url('../fonts/RobotoCondensed-Regular.eot?#iefix') format('embedded-opentype'),
    url('../fonts/RobotoCondensed-Regular.woff') format('woff'),
    url('../fonts/RobotoCondensed-Regular.ttf') format('truetype');
    font-weight: 400;
    font-style: normal;
    }

@font-face {
    font-family: 'Roboto!';
    src:local('Roboto');
    src: url('../fonts/RobotoCondensed-Bold.eot');
    src: url('../fonts/RobotoCondensed-Bold.eot?#iefix') format('embedded-opentype'),
    url('../fonts/RobotoCondensed-Bold.woff') format('woff'),
    url('../fonts/RobotoCondensed-Bold.ttf') format('truetype');
    font-weight: 700;
    font-style: normal;
    }

    a {
    color: black;
    text-decoration: none;
    font-weight: 700;
}
```
